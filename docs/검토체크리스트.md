# 원고 검토 체크리스트 (출판 전)

빌드는 원고를 "있는 그대로" 패키징할 뿐, 내용을 고치지 않는다.
**내용 교정은 반드시 이 검토 단계에서** `manuscript.md`/`book.env`를 직접 고쳐서 한다.

## 절차
1. 자동 점검: `python3 shared/lib/review.py books/<slug>` → ⚠ 항목 확인·수정
2. 아래 사람 눈 체크리스트 확인
3. 모두 통과하면 `book.env`에 `REVIEWED="yes"`, `REVIEW_DATE="YYYY-MM-DD"` 표시
4. 그 다음 빌드(`build.sh`)

## 사람 눈 체크리스트
- [ ] **제목**: 각 장 제목 오타·띄어쓰기 정확 (원본 차례의 붙여쓰기 등 교정)
- [ ] **장 구성**: 차례의 장 수와 본문 장 수 일치, 번호 연속, 누락 없음
- [ ] **본문**: 문단 잘림/중복/순서 오류 없음, 설교 본문이 원문과 일치(요약·누락 금지)
- [ ] **성경 인용**: 도입 성경·인용 구절의 장/절·본문 정확, 절 번호 유지
- [ ] **머리말**: 내용·서명(발행 시기/저자) 정확
- [ ] **판권**: 판(EDITION)·발행일(RELEASE_DATE)·버전(VERSION)·주소·연락처 정확
- [ ] **표지**: 책이름·부제·저자·대표구절·색 적절
- [ ] **이미지**: 로고/교회안내 등 위치·해상도 적절, 깨짐 없음
- [ ] **메타데이터(book.env)**: 제목·저자·태그·소개 정확
- [ ] **깨진 글자**: U+FFFD 등 없음(자동 점검에서도 확인)

## 메모
- 원본 HWP에서 다시 추출(`extract_hwp.py`)하면 `manuscript.md`가 덮어써져 검토 수정이 사라진다.
  → 재추출은 원본이 바뀐 경우에만, 이후 검토를 다시 한다.
